Detailed Description

These settings return information about the library.

Available in all contexts, even before a task model has been loaded.

◆ SNSR_AUTO_FLUSH

#define SNSR_AUTO_FLUSH   "auto-flush"

Recognition pipeline behavior.

Deprecated:
Use snsrPush() and snsrStop() in new code instead.

This boolean value controls whether snsrRun() flushes the recognition pipeline when one (or more) of the input streams report an end-of-file condition.

The default value is 1, which enables automatic flushing on SNSR_RC_EOF. This is appropriate for most applications. Set SNSR_AUTO_FLUSH to 0 when audio is presented to snsrRun() in small segments.

See also
snsrGetInt(), snsrSetInt(), snsrRun(), snsrPush(), snsrStop()

◆ SNSR_LIBRARY_INFO

#define SNSR_LIBRARY_INFO   "library-info"

SDK library information.

Convenient summary of SNSR_NAME, SNSR_VERSION, SNSR_VERSION_DSP, SNSR_ACCELERATION, SNSR_CONFIG_THREAD_SUPPORT, SNSR_CONFIG_STT_SUPPORT, SNSR_LICENSE_EXPIRES, SNSR_LICENSE_WARNING, and SNSR_OSS_COMPONENTS.

Deprecated:
The session handle can be NULL, in which case SNSR_OSS_COMPONENTS will not be included in the summary.

Example:

snsrSession s;
const char *libInfo;
snsrNew(&s);
snsrGetString(s, SNSR_LIBRARY_INFO, &libInfo);
fprintf(stderr, "%s\n", libInfo);
snsrRelease(s);
Examples
live-enroll.c, snsr-edit.c, snsr-eval.c, spot-convert.c, and spot-enroll.c.

◆ SNSR_LICENSE_EXPDATE

#define SNSR_LICENSE_EXPDATE   "license-exp-date"

Library license expiration date.

Returns the license expiration date of the TrulyNatural library in seconds since the Epoch. For production keys, which do not expire, the expiration date is 0. Example:

double e;
time_t expdate;
snsrGetDouble(s, SNSR_LICENSE_EXPDATE, &e);
expdate = (time_t)e;
See also
snsrGetDouble()

◆ SNSR_LICENSE_EXPIRES

#define SNSR_LICENSE_EXPIRES   "license-exp-message"

Library license expiration message.

The returned string is of the form "Library license expires on <date>", or NULL for license keys that do not expire. Example:

const char *expires;
snsrGetString(s, SNSR_LICENSE_EXPIRES, &expires);
if (expires) fprintf(stderr, "%s\n", expires);
See also
snsrGetString()
Examples
live-enroll.c, snsr-edit.c, snsr-eval.c, and spot-enroll.c.

◆ SNSR_LICENSE_WARNING

#define SNSR_LICENSE_WARNING   "license-exp-warn"

Library license expiration warning message.

Use with snsrGetString(). This value is NULL for library license keys that do not expire, or if the expiration date is more than 60 days into the future. For license keys expiring in 60 days or fewer, the returned string will be of the form "License will expire in 37 days.".

Examples
live-enroll.c, snsr-edit.c, snsr-eval.c, and spot-enroll.c.

◆ SNSR_OSS_COMPONENTS

#define SNSR_OSS_COMPONENTS   "oss-components"

Open Source library acknowledgements.

List of Open Source third-party modules linked into the binary.

The returned string lists one module per newline-separated line. Each line contains the name of the library, the license type and a URL to the module license text, separated by tab characters. The TrulyNatural SDK uses permissive software licenses only.

If the binary uses no Open Source modules, the returned string is NULL.

Elide all Open Source modules from a TrulyNatural binary by:

  • Compiling with -DSNSR_OMIT_OSS_COMPONENTS
  • or by using custom initialization with models that do not require these components. See SNSR_PREPARE_SUBSET_INIT, snsr-edit, and How can I reduce application code size?.
See also
snsrGetString(), SNSR_LIBRARY_INFO, Open Source Licenses.

◆ SNSR_PREPARE_SUBSET_INIT

#define SNSR_PREPARE_SUBSET_INIT   "model:ids"

Prepares a SnsrSession for generating custom initialization code.

Set this value to NULL to enable snsrLoad() instrumentation. This is used by snsrSave() with SNSR_FM_SUBSET_INIT to generate custom library initialization code (in C), which references only modules encountered during snsrLoad().

Linking against only a subset of available modules reduces executable size.

This example will create a custom-init.c file. Add this to the build, and recompile with -DSNSR_USE_SUBSET to enable. The linked executable(s) will contain just the modules required to run model-a.snsr and model-b.snsr.

Example:

SnsrSession s;
snsrNew(&s);
snsrSetString(s, SNSR_PREPARE_SUBSET_INIT, NULL);
snsrLoad(s, snsrStreamFromFileName("model-a.snsr", "r"));
snsrLoad(s, snsrStreamFromFileName("model-b.snsr", "r"));
snsrSave(s, SNSR_FM_SUBSET_INIT,
snsrStreamFromFileName("custom-init.c", "w"));
snsrRelease(s);
See also
snsrSave(), snsrSetString(), snsrLoad(), SNSR_FM_SUBSET_INIT
Examples
snsr-edit.c.
