◆ SNSR_INCLUDE_LEADING_SILENCE

#define SNSR_INCLUDE_LEADING_SILENCE   "include-leading-silence"

Include leading silence in VAD output.

Set to 1 to include all audio up to the endpoint in the SNSR_SINK_AUDIO_PCM output stream. Set to 0 to return to the default behavior, which discards leading silence.

If this setting is used with a spot-VAD template (tpl-spot-vad, tpl-spot-vad-lvcsr) the leading silence includes the trigger phrase.

See also
snsrSetInt(), SNSR_SINK_AUDIO_PCM, SNSR_PASS_THROUGH
Examples
live-segment.c.

◆ SNSR_LEADING_SILENCE

#define SNSR_LEADING_SILENCE   "leading-silence"

VAD leading silence time-out, in ms.

The Voice Activity Detector will invoke the SNSR_SILENCE_EVENT handler if no speech is detected during the first SNSR_LEADING_SILENCE ms of processed audio.

See also

◆ SNSR_LIMIT_EVENT

#define SNSR_LIMIT_EVENT   "^limit"

Maximum recording reached VAD event.

Raised when SNSR_MAX_RECORDING ms of speech has been processed by the Voice Activity Detector before a trailing-silence endpoint is found. Use SNSR_RES_END_MS or SNSR_RES_END_SAMPLE to retrieve the endpoint relative to the beginning of SNSR_SOURCE_AUDIO_PCM.

See also

◆ SNSR_LOOP

#define SNSR_LOOP   "loop"

Control template looping behavior In tpl-spot-sequential, setting this value to 1 changes when the listening focus returns to SNSR_SLOT_0.

Instead of immediately returning to SNSR_SLOT_0 after a spot in SNSR_SLOT_1, it resets the expiration timer, and only a timeout returns to SNSR_SLOT_0.

This allows for a wake word followed by zero or more commands from a command set. The default behavior is to allow at most one command before requiring another wake word utternace.

See also
snsrSetInt(), tpl-spot-sequential

◆ SNSR_PASS_THROUGH

#define SNSR_PASS_THROUGH   "pass-through"

VAD audio pass-through behavior.

If set to 0, no audio from SNSR_SOURCE_AUDIO_PCM will be passed through to SNSR_SINK_AUDIO_PCM. The begin- and endpoint handlers will still be invoked. The default value (1) passes speech-detected samples to SNSR_SINK_AUDIO_PCM.

See also
snsrSetInt(), SNSR_INCLUDE_LEADING_SILENCE
Examples
snsr-eval.c.

◆ SNSR_SILENCE_EVENT

#define SNSR_SILENCE_EVENT   "^silence"

No speech detected event.

Raised if no speech is detected within SNSR_LEADING_SILENCE ms from the start of SNSR_SOURCE_AUDIO_PCM (adjusted by SNSR_SKIP_TO_MS or SNSR_SKIP_TO_SAMPLE).

See also
snsrSetHandler(), SNSR_LEADING_SILENCE
Examples
push-audio.c, and snsr-eval.c.

◆ SNSR_SKIP_TO_MS

#define SNSR_SKIP_TO_MS   "skip-to-ms"

VAD initial ignore duration, in ms.

Ignore the first SNSR_SKIP_TO_MS ms of the SNSR_SOURCE_AUDIO_PCM input stream. Use this runtime setting to skip over a trigger phrase included in the source audio. The default is to process all audio.

See also
snsrSetDouble(), SNSR_SKIP_TO_SAMPLE

◆ SNSR_SKIP_TO_SAMPLE

#define SNSR_SKIP_TO_SAMPLE   "skip-to-sample"

VAD initial ignore duration, in samples.

Ignore the first SNSR_SKIP_TO_SAMPLE samples of the SNSR_SOURCE_AUDIO_PCM input stream. Use this runtime setting to skip over a trigger phrase included in the source audio. The default is to process all audio.

See also
snsrSetDouble(), SNSR_SKIP_TO_MS
