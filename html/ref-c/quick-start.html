Build with CMake (Linux, macOS, and Windows)

Samples

$ cmake -S "/Users/sankalp/Sensory/TrulyNaturalSDK/7.4.0/sample/c" -B build-sample
$ cmake --build build-sample --parallel --config Release
$ cmake --install build-sample

Then find the sample executables in build-sample/bin

Add the TrulyNatural SDK to your application

Add to your CMakeLists.txt:

list(APPEND CMAKE_MODULE_PATH "/Users/sankalp/Sensory/TrulyNaturalSDK/7.4.0")
include(SnsrLibrary)

and to each of your executable targets:

target_link_libraries(your-target SnsrLibrary)

If you do not want to include Open Source Software components, and therefore no Speech-To-Text support, use SnsrLibraryOmitOSS intead:

target_link_libraries(your-target SnsrLibraryOmitOSS)

Add the TrulyNatural SDK to your application

  • Add -I/Users/sankalp/Sensory/TrulyNaturalSDK/7.4.0/include to your compiler flags,
  • Add -L/Users/sankalp/Sensory/TrulyNaturalSDK/7.4.0/lib/<platform> to the linker flags, and
  • Linux: Add -lsnsr -lasound -lpthread -lm -ldl -lstdc++ to the list of libraries to link against.
  • macOS: Add -lsnsr -framework AudioToolbox -framework Accelerate -framework CoreFoundation -framework Foundation -lm -lstdc++ to the linker flags.

Example:

$ gcc -dumpmachine
x86_64-linux-gnu
$ gcc -o live-enroll \
-I/Users/sankalp/Sensory/TrulyNaturalSDK/7.4.0/include\
-L/Users/sankalp/Sensory/TrulyNaturalSDK/7.4.0/lib/x86_64-linux-gnu\
src/live-enroll.c -lsnsr -lasound -lpthread -lm -ldl -lstdc++

iOS sample apps

iOS applications amples are in sample/ios/ in the TrulyNatural installation directory.

The TrulyNatural SDK requires linking against the Accelerate and AudioToolbox frameworks.

PhraseSpot

This Swift application runs a phrase spotter and shows the results in a text window. It links against the C TrulyNatural SDK.

Build and run

Open sample/ios/PhraseSpot/PhraseSpot.xcodeproj in Xcode, then choose Run from the Product menu.

To run on a real device, select your Team in "General" > "Signing" and change "Identity" > "Bundle Identifier" to match your development domain.

The app starts listening for "hello blue genie" upon startup. Say this trigger phrase to see the recognizer response.

TrulyNatural configuration

To add the TrulyNatural SDK to an existing iOS app:

  • Add a Bridging header, ProjectName-Bridging-Header.h, that includes
    #import <snsr.h>
  • Select the application target.
    • In "Capabilities", add "Inter-App Audio Capability".
    • In "Info" > "Custom iOS Target Properties", add an NSMicrophoneUsageDescription string.
    • In "Build Settings",
      • Add ${SNSR_ROOT}/include to the "Header Search Paths".
      • Add ${SNSR_ROOT}/lib/ios to the "Library Search Paths".
      • Add -lsnsr to "Other Linker Flags".
    • In "General" > "Frameworks, Libraries, and Embedded Content",
      • Add Accelerate.framework
      • Add AudioToolbox.framework
  • Select the project.
    • Select the "Editor > Add Build Setting > Add User-Defined Setting" menu entry.
    • Add SNSR_ROOT. Set this variable to the TrulyNatural SDK installation directory, /Users/sankalp/Sensory/TrulyNaturalSDK/7.4.0.
