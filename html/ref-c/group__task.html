Detailed Description

These settings are task-specific, and valid only after a task has been loaded into as session.

Macro Definition Documentation

◆ SNSR_MODEL_LICENSE_EXPDATE

#define SNSR_MODEL_LICENSE_EXPDATE   "model-license-exp-date"

Model license expiration date.

Returns the license expiration date of the most recently loaded model in seconds since the Epoch, or 0 if no model is loaded. For production keys, which do not expire, the expiration date is 0. Example:

double e;
time_t expdate;
snsrGetDouble(s, SNSR_MODEL_LICENSE_EXPDATE, &e);
expdate = (time_t)e;
See also
snsrGetDouble()

◆ SNSR_MODEL_LICENSE_EXPIRES

#define SNSR_MODEL_LICENSE_EXPIRES   "model-license-exp-message"

Model license expiration message.

Returns an expiration message string for the most recently loaded model, or NULL if no model is loaded. The returned string is of the form "Model license expires on <date>", or NULL for model license keys that do not expire. Example:

const char *expires;
snsrGetString(s, SNSR_MODEL_LICENSE_EXPIRES, &expires);
if (expires) fprintf(stderr, "%s\n", expires);
See also
snsrGetString()
Examples
live-enroll.c, snsr-edit.c, snsr-eval.c, and spot-enroll.c.

◆ SNSR_MODEL_LICENSE_WARNING

#define SNSR_MODEL_LICENSE_WARNING   "model-license-exp-warn"

Model license expiration warning message.

Use with snsrGetString(). This value is NULL for model license keys that do not expire, if the expiration date is more than 60 days into the future, or if no model is loaded into the session. For license keys expiring in 60 days or fewer, the returned string will be of the form "License will expire in 37 days.".

Examples
live-enroll.c, snsr-edit.c, snsr-eval.c, spot-convert.c, and spot-enroll.c.

◆ SNSR_MODEL_NAME

#define SNSR_MODEL_NAME   "model-name"

Source model name.

The name of the model file used to create:

  • ROM-able C code with snsrSave() and SNSR_FM_SOURCE,
  • a model in one of the supported embedded formats with SNSR_EMBEDDED_HEADER_STREAM.

This value is included in the comment header of the generated code, for information only.

Examples
snsr-edit.c, and spot-convert.c.

◆ SNSR_PUSH_BUFFER_SIZE

#define SNSR_PUSH_BUFFER_SIZE   "push-buffer-size"

The size of the internal ring buffers used by snsrPush() If snsrPush() is used with a SNSR_PUSH_DURATION_LIMIT, processing will require deferral if the duration limit is reached.

In this case, snsrPush() will allocate a ring buffer to hold these data. This setting configures the size of this buffer, in bytes.

The default buffer size is sufficient to defer up to 250 ms of audio data.

See also
snsrSetInt(), snsrGetInt(), SNSR_PUSH_DURATION_LIMIT, SNSR_RES_PUSH_BUFFER_BACKLOG

◆ SNSR_PUSH_DURATION_LIMIT

#define SNSR_PUSH_DURATION_LIMIT   "push-duration-limit"

Sets a limit to the maximum processing time snsrPush() should consume.

This setting is the maximum number of miliseconds any call to snsrPush() should spend processing data before returning control to the caller.

The default value is 0, which disables the processing limit.

Note
This requires a valid real-time clock function, see SNSR_CONFIG_CLOCK_FUNC.

You should use a push-duration-limit if:

  • You're using snsrPush().
  • You collect live audio on the same thread as the recognizer.
  • You will drop audio packets if you don't return from snsrPush() before the next packet is available.
  • SNSR_PUSH_DURATION_LIMIT adds a cap to the amount of CPU used in each call to snsrPush(). This requires and allocates an additional input ring buffer.

If you have a separate thread, or interrupt-driven live audio recording and you want to maximize throughput, increase the size of the audio ring buffer instead of using a SNSR_PUSH_DURATION_LIMIT. Recommendations:

  • Use 15 ms audio chunks.
  • The audio recording buffer size determines the longest time the average recognizer throughput can fall behind real time.
  • With a a 30 ms buffer only two 15 ms block fit, which means that every SDK processing call must return within 15 ms, or we'll lose a block or partial block.
  • Using a 300 ms buffer relaxes this. 20 blocks mean that we can fall up to 18 blocks (270 ms) behind before losing audio.
See also
snsrSetDouble(), snsrPush(), SNSR_CONFIG_CLOCK_FUNC, SNSR_PUSH_BUFFER_SIZE, SNSR_RES_PUSH_BUFFER_BACKLOG

◆ SNSR_RES_PUSH_BUFFER_BACKLOG

#define SNSR_RES_PUSH_BUFFER_BACKLOG   "push-buffer-backlog"

Reports the number of bytes of deferred snsrPush() data.

If snsrPush() is used with a SNSR_PUSH_DURATION_LIMIT, this setting reports the number of bytes deferred for processing in subsequent calls to snsrPush().

See also

◆ SNSR_TAG_IDENTIFIER

#define SNSR_TAG_IDENTIFIER   "tag-identifier"

Exported identifier in ROM C code.

When text segment C code is created with snsrSave() and SNSR_FM_SOURCE, this setting specifies the name of the exported C data structure.

The value must start with an ASCII alphabetic character or _, and contain only alphanumerics and _.

◆ SNSR_TASK_TYPE_AND_VERSION_LIST

#define SNSR_TASK_TYPE_AND_VERSION_LIST   "task-type-and-version-list"

A list of task names and versions.

When used with snsrRequire(), the value argument must be a semicolon-separated list of SNSR_TASK_TYPE and SNSR_TASK_VERSION values. This list must have at least one element.

A task will match the requirement if one of the SNSR_TASK_TYPE fields match, and the corresponding SNSR_TASK_VERSION is satisfied.

If no SNSR_TASK_TYPE matches, snsrRequire() returns SNSR_RC_REQUIRE_MISMATCH. If a SNSR_TASK_TYPE matches, but the associated SNSR_TASK_VERSION is not satisfied, snsrRequire() returns SNSR_RC_VERSION_MISMATCH.

Example:

snsrRequire(session, SNSR_TASK_TYPE_AND_VERSION_LIST,
SNSR_PHRASESPOT " ~0.5.0 || 1.0.0;"
SNSR_LVCSR " 1.0.0");
See also
snsrRequire()
Examples
push-audio.c, and snsr-eval.c.
