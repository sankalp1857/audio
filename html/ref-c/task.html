Phrase Spotter

Raises an event when a preconfigured phrase is spotted in the input audio stream. Multiple phrases can be active at the same time. The Spotter Enrollment task provides a convenient way to enroll user-defined phrases, which can also be used for speaker verification.

Behavior

  • Reads audio data from the input stream SNSR_SOURCE_AUDIO_PCM.
  • Invokes SNSR_SAMPLES_EVENT.
  • If one of the phrases in the model is spotted, invokes SNSR_RESULT_EVENT.
  • Continues processing until SNSR_RC_EOF is encountered on the input stream, or one of the event handlers return a code other than SNSR_RC_OK.

Example

  • See snsr-eval.c and spot-convert.c.

Spotter Enrollment

Fixed and User-Defined Trigger enrollment details.

Behavior

Interactive mode (the default)

SNSR_INTERACTIVE_MODE == 1

  • Invokes SNSR_RESUME_EVENT.
  • Reads audio data from the input stream SNSR_SOURCE_AUDIO_PCM.
  • Invokes SNSR_SAMPLES_EVENT.
  • Repeats until an enrollment segment is detected.
  • If SNSR_USER is not set, invokes SNSR_NEXT_EVENT.
  • Invokes SNSR_PAUSE_EVENT.
  • Checks enrollment recording, invokes SNSR_PASS_EVENT if good, SNSR_FAIL_EVENT if not.
  • Repeats until the required number of enrollments have been recorded.
  • Validates all enrollments, invokes SNSR_PASS_EVENT if good, SNSR_FAIL_EVENT if not.
  • Invokes SNSR_NEXT_EVENT. If a new user is encountered repeats from the top.
  • Invokes SNSR_ENROLLED_EVENT.
  • Starts adaptation, invoking SNSR_PROG_EVENT periodically.
  • Invokes SNSR_DONE_EVENT.

Batch mode

SNSR_INTERACTIVE_MODE == 0

  • Invokes SNSR_RESUME_EVENT.
  • Reads audio data from the input stream SNSR_SOURCE_AUDIO_PCM.
  • Invokes SNSR_SAMPLES_EVENT.
  • Repeats until an enrollment segment is detected.
  • If SNSR_USER is not set, invokes SNSR_NEXT_EVENT.
  • Invokes SNSR_PAUSE_EVENT.
  • Checks enrollment recording, invokes SNSR_PASS_EVENT if good, SNSR_FAIL_EVENT if not.
  • Invokes SNSR_RESUME_EVENT.
  • Repeats from the top until SNSR_SOURCE_AUDIO_PCM reaches SNSR_RC_EOF.
  • Validates all enrollments, invokes SNSR_PASS_EVENT if good, SNSR_FAIL_EVENT if not.
  • Repeats until SNSR_USER is set to NULL.
  • Invokes SNSR_ENROLLED_EVENT.
  • Starts adaptation, invoking SNSR_PROG_EVENT periodically.
  • Invokes SNSR_DONE_EVENT.

Example

  • See spot-enroll.c

Voice Activity Detector

Segments a single speech-like region from an audio stream.

Behavior

  • Reads audio data from the input stream SNSR_SOURCE_AUDIO_PCM.
  • Invokes SNSR_SAMPLES_EVENT.
  • Invokes SNSR_BEGIN_EVENT when speech is first detected.
  • Invokes an endpoint event (SNSR_END_EVENT, SNSR_LIMIT_EVENT, or SNSR_SILENCE_EVENT) when the final speech point has been determined.
  • Continues processing until SNSR_RC_EOF is encountered on the input stream, or one of the event handlers return a code other than SNSR_RC_OK.
    • The default behavior for endpoint events is to return SNSR_RC_STOP.
  • Between the SNSR_BEGIN_EVENT and the endpoint event audio samples from SNSR_SOURCE_AUDIO_PCM are copied to SNSR_SINK_AUDIO_PCM.
    • See SNSR_INCLUDE_LEADING_SILENCE, SNSR_PASS_THROUGH.
  • Endpoint events restart the VAD using the same settings.

Example

  • See live-segment.c

Settings

  • Task type validation
    • snsrRequire(session, SNSR_TASK_TYPE, SNSR_VAD)
    • SNSR_TASK_VERSION == 1.0.0.
  • Configuration settings
    • SNSR_BACKOFF, SNSR_HOLD_OVER, SNSR_INCLUDE_LEADING_SILENCE, SNSR_LEADING_SILENCE, SNSR_MAX_RECORDING, SNSR_PASS_THROUGH, SNSR_TRAILING_SILENCE, SNSR_VAD.
  • Runtime settings
    • SNSR_SKIP_TO_MS, SNSR_SKIP_TO_SAMPLE.
    • Handlers
      • The runtime and result settings below are available only during the execution of the handler functions.
      • SNSR_BEGIN_EVENT
        • Results: SNSR_RES_BEGIN_MS, SNSR_RES_BEGIN_SAMPLE.
      • SNSR_END_EVENT, SNSR_LIMIT_EVENT
        • Results: SNSR_RES_BEGIN_MS, SNSR_RES_BEGIN_SAMPLE, SNSR_RES_END_MS, SNSR_RES_END_SAMPLE.
      • SNSR_SAMPLES_EVENT
        • Result: SNSR_RES_SAMPLES.
      • SNSR_SILENCE_EVENT
        • No speech detected: No segmentation results available.
    • Streams
      • SNSR_SOURCE_AUDIO_PCM, SNSR_SINK_AUDIO_PCM.

Phrase Spotter with VAD

This task is a combination of Phrase Spotter and Voice Activity Detector. The audio directly following the end of the spotted phrase is gated by the VAD, which provides start- and end-pointing.

Use this task type for spotter-to-search applications, where the VAD audio is sent to a third party recognition engine.

Behavior

  • Reads audio data from the input stream SNSR_SOURCE_AUDIO_PCM.
  • Invokes SNSR_SAMPLES_EVENT.
  • If one of the phrases in the model is spotted:
    • Invokes SNSR_RESULT_EVENT.
    • Invokes SNSR_LISTEN_BEGIN_EVENT.
    • Processes audio following the phrase with the VAD.
      • Invokes SNSR_BEGIN_EVENT when speech is first detected.
      • Invokes an endpoint event (SNSR_END_EVENT, SNSR_LIMIT_EVENT, or SNSR_SILENCE_EVENT) when the final speech point has been determined.
      • Between the SNSR_BEGIN_EVENT and the endpoint event audio samples from SNSR_SOURCE_AUDIO_PCM are copied to SNSR_SINK_AUDIO_PCM.
      • See SNSR_INCLUDE_LEADING_SILENCE, SNSR_PASS_THROUGH.
    • Invokes SNSR_LISTEN_END_EVENT and switches audio processing back to the phrase spotter.
  • Continues processing until SNSR_RC_EOF is encountered on the input stream, or one of the event handlers return a code other than SNSR_RC_OK.

Example

With a phrase spotter for "Alexa", the phrase "Alexa will it rain in Portland tomorrow?" will provide the audio for "will it rain in Portland tomorrow?" in the SNSR_SINK_AUDIO_PCM stream. The application can register handlers to receive SNSR_BEGIN_EVENT and SNSR_END_EVENT callbacks.

Settings

This task includes all settings from Phrase Spotter and Voice Activity Detector.

  • Additional runtime settings
    • SNSR_SINK_AUDIO_PCM contains the audio samples selected by the VAD.

Large Vocabulary Continuous Speech Recognizer

Converts speech to text. Raises events when partial and final hypotheses are available.

Task version 1.1.0 and later includes support for a lightweight experimental VAD. This uses the <s> and </s> delimiters at the start and end of the recoginition results, and does not consider the speech energy levels.

Behavior

  • Reads audio data from the input stream SNSR_SOURCE_AUDIO_PCM.
  • Invokes SNSR_SAMPLES_EVENT.
  • Invokes SNSR_PARTIAL_RESULT_EVENT with the current best hypothesis once every SNSR_PARTIAL_RESULT_INTERVAL ms. This is disabled when SNSR_PARTIAL_RESULT_INTERVAL is set to 0.
  • When SNSR_RC_EOF is encountered in the input stream, or when a Voice Activity Detector endpoint is detected:
    • If any NLU semantic tags are included in the word-level results, invokes SNSR_NLU_SLOT_EVENT for each top-level tag encountered.
    • Invokes SNSR_RESULT_EVENT with the final hypothesis.
  • Continues processing until SNSR_RC_EOF is encountered on the input stream, or one of the event handlers return a code other than SNSR_RC_OK.

Example

  • See snsr-eval.c.

Settings

  • Task type validation
    • snsrRequire(session, SNSR_TASK_TYPE, SNSR_LVCSR)
    • SNSR_TASK_VERSION == 1.1.0.
  • Configuration settings

    • SNSR_BACKOFF, SNSR_HOLD_OVER, SNSR_LEADING_SILENCE, SNSR_MAX_RECORDING, SNSR_TRAILING_SILENCE.
  • Runtime settings
    • Handlers
      • The runtime and result settings below are available only during the execution of the handler functions.
      • SNSR_BEGIN_EVENT
        • Results: SNSR_RES_BEGIN_MS, SNSR_RES_BEGIN_SAMPLE.
      • SNSR_END_EVENT, SNSR_LIMIT_EVENT
        • Results: SNSR_RES_BEGIN_MS, SNSR_RES_BEGIN_SAMPLE, SNSR_RES_END_MS, SNSR_RES_END_SAMPLE.
      • SNSR_SILENCE_EVENT

      • SNSR_NLU_SLOT_EVENT

      • SNSR_RESULT_EVENT
        • Iterators: SNSR_PHRASE_LIST, SNSR_WORD_LIST.
        • Results: SNSR_RES_BEGIN_MS, SNSR_RES_BEGIN_SAMPLE, SNSR_RES_END_MS, SNSR_RES_END_SAMPLE, SNSR_RES_TEXT.
      • SNSR_SAMPLES_EVENT
        • Result: SNSR_RES_SAMPLES.
    • Iterators
      • SNSR_NLU_SLOT_LIST.
        • Results: SNSR_RES_NLU_SLOT_COUNT, SNSR_RES_NLU_SLOT_NAME, SNSR_RES_NLU_SLOT_VALUE.

        • Results: SNSR_RES_BEGIN_MS, SNSR_RES_BEGIN_SAMPLE, SNSR_RES_END_MS, SNSR_RES_END_SAMPLE, SNSR_RES_TEXT.

        • Results: SNSR_RES_BEGIN_MS, SNSR_RES_BEGIN_SAMPLE, SNSR_RES_END_MS, SNSR_RES_END_SAMPLE, SNSR_RES_TEXT.

Task Templates

Task templates are models with custom behavior, and one or more empty slots that can be filled with phrase spotter modules. Slots can be filled during development using the snsr-edit tool, or at runtime with the snsrSetStream() API call.

This release includes these templates:

tpl-spot-concurrent

  • Runs the spotters in SNSR_SLOT_0 and SNSR_SLOT_1 at the same time, in the same thread.
  • The combined model is a Phrase Spotter, and can be used in any application without API changes.
    • Most combined model settings affect both embedded spotters. Settings SNSR_OPERATING_POINT, SNSR_OPERATING_POINT_LIST, and SNSR_RES_AVAILABLE_POINT are not available in this template.
    • Change individual settings at runtime by prefixing the setting name with the slot:
      /* Set the operating point for spotter 0 only. */
      snsrSetInt(session, SNSR_SLOT_0 SNSR_OPERATING_POINT, 7);
  • Note that the two phrase spotters are independent, and can produce results that overlap in time. For production use Sensory recommends custom multi-phrase phrase spotters instead. These have improved false reject / false accept performance.

tpl-spot-debug

  • Use this for spotter debug logging.
  • Runs the spotter in SNSR_SLOT_0.
  • Requires a path to the output logfile in SNSR_DEBUG_LOG_FILE.
  • SNSR_INCLUDE_MODEL controls whether the log file includes a copy of the original task model. This is enabled by default.
  • The combined model is a Phrase Spotter, and can be used in any application without API changes.
  • Logs record:
    • SDK version and acceleration in play,
    • the spotter model being used,
    • audio, and
    • event callbacks.
  • Extract text, model and audio data from the log file with the snsr-log-split utility, see snsr-log-split.

tpl-spot-select

  • Use SNSR_SLOT to select either the spotter in SNSR_SLOT_0 or in SNSR_SLOT_1.
    • ‡ Note that tpl-spot-vad-lvcsr an tpl-vad-lvcsr conform to the Phrase Spotter API, and can be used in either slot in this template.
  • Use this template to reduce the model size when an application uses variants of the same recognizer in different contexts. If, for example, an application uses both tpl-spot-vad-lvcsr and tpl-vad-lvcsr with the same LVCSR recognizer, slotting these into tpl-spot-select will roughly halve the total size required. The reason being that model files contain just one copy of each unique object.
  • The template model is a Phrase Spotter, and can be used in any application that supports this type without API changes.

tpl-spot-sequential

  • With SNSR_LOOP set to 0 (the default): Runs the spotter in SNSR_SLOT_0 until it spots, then runs SNSR_SLOT_1 until it spots, or the SNSR_LISTEN_WINDOW timeout expires, then returns to the spotter in SNSR_SLOT_0.
  • With SNSR_LOOP set to 1: Runs the spotter in SNSR_SLOT_0 until it spots, then runs SNSR_SLOT_1 until the SNSR_LISTEN_WINDOW timeout expires, then returns to the spotter in SNSR_SLOT_0. Resets the expiration timer every time SNSR_SLOT_1 spots.
  • Use this to listen for a trigger phrase which is followed by a command. For example, "Hello blue genie, play music."
  • The combined model is a Phrase Spotter, and can be used in any application without API changes.
    • Combined model settings refer to the spotter in SNSR_SLOT_1.
    • Change settings for SNSR_SLOT_0 by prefixing the setting name with SNSR_SLOT_0.
  • The SNSR_LISTEN_BEGIN_EVENT is raised just before audio focus switches to SNSR_SLOT_1, and SNSR_LISTEN_END_EVENT fires just before audio focus switches back to SNSR_SLOT_0. If there's no SNSR_RESULT_EVENT between these events it is an indication that the SNSR_LISTEN_WINDOW on SNSR_SLOT_1 has timed out.

tpl-spot-vad

  • Runs the spotter in SNSR_SLOT_0 until it spots, then does start- and endpoint detection on the subsequent audio stream using a Voice Activity Detector.
  • Use this for trigger-to-search applications.
  • The combined model is a Phrase Spotter with VAD.
  • Set SNSR_INCLUDE_LEADING_SILENCE to 1 to include the trigger audio in the VAD audio output stream.

tpl-spot-vad-lvcsr

  • Runs the spotter in SNSR_SLOT_0 until it spots, does start- and endpoint detection on the subsequent audio stream using a Voice Activity Detector, and sends this segmented audio to the large-vocabulary recognizer in SNSR_SLOT_1.
  • SNSR_PARTIAL_RESULT_EVENT and SNSR_RESULT_EVENT refer to the large-vocabulary recognizer in SNSR_SLOT_1. If you need direct access to spotter results, prefix these settings with SNSR_SLOT_0.
  • The combined model is a Phrase Spotter.
  • Set SNSR_INCLUDE_LEADING_SILENCE to 1 to pass the trigger audio to the LVCSR recognizer.

tpl-vad-lvcsr

  • Does start- and endpoint detection on SNSR_SOURCE_AUDIO_PCM, using a Voice Activity Detector, then sends this segmented audio to the large-vocabulary recognizer in SNSR_SLOT_0.
  • The combined model is a Phrase Spotter.
