◆ snsrNew

#define snsrNew (   s)    snsrNewIncludeOSS((s), SNSR_VERSION)

Creates a new SnsrSession handle.

Session handles contain the entire state for a task.

If a SnsrSession handle has to be shared across threads, all calls into this library that use the handle must be protected by application-level thread synchronization calls. The SnsrSession API calls are not re-entrant.

Best practice is to use a single thread per handle. Using multiple handles per thread is fully supported.

snsrNew() is implemented as a macro, which validates the included snsr.h against the one that the archive library was built with. If these do not match, snsrNew() fails and returns SNSR_RC_LIBRARY_HEADER.

Parameters
[out]sa pointer to an allocated SnsrSession handle.
Returns
SNSR_RC_OK for success, any other value indicates failure. If s is not NULL a detailed error message can be retrieved with snsrErrorDetail().
See also
snsrDup()
Examples
live-enroll.c, live-segment.c, live-spot-stream.c, live-spot.c, push-audio.c, snsr-edit.c, snsr-eval.c, spot-convert.c, spot-data-stream.c, spot-data.c, and spot-enroll.c.

◆ SnsrSession

typedef struct SnsrSession_* SnsrSession

Opaque session handle.

This is the primary handle for SDK operations. Session handles contain the entire state for a task.

Functions operating on the same SnsrSession instance handle must be protected from re-entrant calls with application-level locks.

Synchronization and locking is not needed if each SnsrSession instance is accessed from a single thread only.

SNSR_API void snsrClearRC ( SnsrSession  s)

Clears the SnsrSession error code.

Once a SnsrSession error occurs (i.e. a return code other than SNSR_RC_OK is reported), all library calls on the session handle will immediately return with the same error code.

Call this function to clear the error detail and reset the code to SNSR_RC_OK.

Parameters
[in]ssession handle.

◆ snsrDup()

SNSR_API SnsrRC snsrDup ( SnsrSession  s,
SnsrSession *  dst 
)

Duplicates a session handle.

Creates a new SnsrSession handle that is a clone of a source handle. This function is roughly equivalent to:

SnsrStream b = snsrStreamFromBuffer(1024, 1073741824);
snsrSave(s, SNSR_FM_CONFIG, b);
snsrLoad(*dst, b);
snsrRelease(b);

The new SnsrSession dst contains all of the configuration from s, but none of the runtime settings. Immutable configuration settings (such as acoustic models) are shared between s and dst. Using snsrDup() to create two runtime instances of a task will therefore use less memory than loading the same task data into two SnsrSession handles.

Parameters
[in]ssource session handle.
[out]dstpointer to an allocated SnsrSession handle which will receive a clone of s.
Returns
SNSR_RC_OK for success, any other value indicates failure.

◆ snsrErrorDetail()

SNSR_API const char* snsrErrorDetail ( SnsrSession  s)

Retrieves a detailed session error message.

This human-readable error message should be used for display or logging only. The content of the message is system-specific. Do not parse this to determine program flow, use the SnsrRC return code instead.

Parameters
[in]ssession handle.
Returns
A detailed message describing the most recent reason for failure. The memory pointed to is owned by this library and must not be released. It is not reference-counted, calling snsrRetain() or snsrRelease() on this handle will panic the heap allocator. The handle remains valid only until the next API call on s.

◆ snsrGetStream()

SNSR_API SnsrRC snsrGetStream ( SnsrSession  s,
const char *  key,
SnsrStream *  stream 
)

Retrieves a stream handle from a runtime session setting.

Returns the SnsrStream value set for key in SnsrSession s. The key and stream arguments must not be NULL.

By default, the returned stream SnsrStream handle is valid only until the next library call on the s SnsrSession handle. This validity can be extended by retaining the handle with snsrRetain(). Be sure to (eventually) release retained handles or a memory leak will result.

See Task Descriptions for a list of keys and the contexts in which they are available.

If key refers to a slot in one of the Task Templates, stream will contain a serialized version of that slot in SNSR_FM_CONFIG format.

Note
This function retrieves only runtime settings (such as recognition audio or enrolled phrase spot modules) and models from template slots. It cannot be used to get the stream handles set with snsrSetStream().
Parameters
[in]ssession handle.
[in]keysetting key.
[out]streamSnsrStream handle associated with key.
Returns
SNSR_RC_OK for success, any other value indicates failure.
See also
snsrSave(), SNSR_SLOT_0, SNSR_SLOT_1
Examples
live-enroll.c, snsr-edit.c, spot-convert.c, and spot-enroll.c.

SNSR_API SnsrRC snsrLoad ( SnsrSession  s,

)

Loads a task model into a session.

Loads a Task Descriptions or configuration settings from a SnsrStream into SnsrSession s. The stream data must be in the format produced by snsrSave(). This function retains a reference to inputStream on entry and releases it on exit. inputStream will be opened, but will not be closed explicitly. Streams are closed before deallocation, so inputStream would be closed if no other references to it are held.

Parameters
[in]ssession handle.
[in]inputStreamreadable SnsrStream that contains a task model.
Returns

◆ snsrPush()

SNSR_API SnsrRC snsrPush ( SnsrSession  s,
const char *  name,
const void *  data,
size_t  sizeInBytes 
)

Processes data in a session.

Adds data to the name input SnsrStream for SnsrSession s, then process these and other buffered stream data. Invokes User Callbacks registered in the session and (if defined) writes to output streams.

Processing will continue until:

  • All sizeInBytes of data are processed.
  • Or one of the invoked callbacks returns an error code, which snsrPush() will return. SNSR_RC_INTERRUPTED, SNSR_RC_REPEAT, SNSR_RC_SKIP, SNSR_RC_STOP or SNSR_RC_TIMED_OUT should be used to indicate that the operation was stopped on request.
  • Or the SNSR_PUSH_DURATION_LIMIT timeout is reached.

User Callbacks are invoked on the thread that snsrPush() was called on. The processing loop is single-threaded, so it stalls while waiting for user functions to return.

If SNSR_PUSH_DURATION_LIMIT is used, snsrPush() has to defer processing when the duration limit is reached. In this case, data are added to an internal ring buffer. snsrPush() will report SNSR_RC_NOT_ENOUGH_SPACE if no ring ring buffer space remains for this deferral. To address such a failure, you could:

  • Increase the SNSR_PUSH_BUFFER_SIZE
  • Increase the SNSR_PUSH_DURATION_LIMIT
  • Use a model with lower CPU requirements.
Parameters
[in]ssession handle.
[in]namethe name of an input stream, such as SNSR_SOURCE_AUDIO_PCM.
[in]dataread-only pointer to data, typically audio samples.
[in]sizeInBytesthe amount of data available in data, in bytes.
Returns
SNSR_RC_OK for success, any other value indicates failure.
See also

◆ snsrRequire()

SNSR_API SnsrRC snsrRequire ( SnsrSession  s,
const char *  key,
const char *  value 
)

Validates task requirements.

Checks that the value for the key, for the task loaded into s, matches the required value value.

If key is SNSR_TASK_VERSION, this function uses semantic version precedence to determine whether the task version meets the value version requirement.

If key is SNSR_TASK_TYPE_AND_VERSION_LIST, version is a list of acceptable SNSR_TASK_TYPE and SNSR_TASK_VERSION tuples separated by ;

value is a semantic version range:

  • A range is composed of one or more sets, joined by " || ". A version matches a range if, and only if, every comparator in at least one of the " || " -separated sets is satisfied by the version.
  • A set is a list of comparators separated by whitespace. A set is satisfied by the intersection of all the comparators it includes.
  • A comparator is an operator followed by a version.
  • operators include:
    • =, <, <=, >, >=: range comparisons.
    • ~: taskVersion >= value && taskVersion < M where M is the next major version number.
    • ^: taskVersion >= value && taskVersion < M where M is the next major version number, skipping over zeros from the left. For taskVersion >= 1.0.0 this is equivalent to the ~ operator. ^0.minor.patch allows only patch updates, e.g. ^0.1.1 will accept 0.1.2 but not 0.2.0.
    • The default behavior when no operator is specified is ^.

This function is used as a sanity check, to verify that the task model loaded is of the correct type, and that it has a compatible version.

Example:

snsrLoad(s, snsrStreamFromFileName("spotter.snsr", "r"));
snsrRequire(s, SNSR_TASK_TYPE, SNSR_PHRASESPOT);
snsrRequire(s, SNSR_TASK_VERSION, "1.0.0");
snsrRequire(s, SNSR_TASK_VERSION, "0.5.0 || 1.0.0");
snsrRequire(s, SNSR_TASK_TYPE_AND_VERSION_LIST,
SNSR_PHRASESPOT " ~0.5.0 || 1.0.0;"
SNSR_PHRASESPOT_VAD " 1.2.3");
Parameters
[in]ssession handle.
[in]keythe name of a setting to validate.
[in]valuestring to match the key setting against.
Returns
SNSR_RC_OK for success, any other value indicates failure.
Examples
live-enroll.c, live-segment.c, live-spot-stream.c, live-spot.c, push-audio.c, snsr-edit.c, snsr-eval.c, spot-convert.c, spot-data-stream.c, spot-data.c, and spot-enroll.c.

◆ snsrRun()

SNSR_API SnsrRC snsrRun ( SnsrSession  s)

Processes stream data in a session.

Enters the main SnsrSession processing loop. This function will process data read from input streams, invoke User Callbacks registered in the session and (if defined) write to output streams.

Processing will continue until:

  • One of the data streams reach SNSR_RC_EOF, in which case snsrRun() returns SNSR_RC_STREAM_END.
  • Or one of the invoked callbacks returns an error code, which snsrRun() will return. SNSR_RC_INTERRUPTED, SNSR_RC_REPEAT, SNSR_RC_SKIP, SNSR_RC_STOP or SNSR_RC_TIMED_OUT should be used to indicate that the operation was stopped on request.

User Callbacks are invoked on the thread that snsrRun() was called on. The processing loop is single-threaded, so it stalls while waiting for user functions to return.

snsrRun() will flush the internal recognition pipeline an input stream reaches SNSR_RC_EOF, unless SNSR_AUTO_FLUSH is set to 0.

[in]ssession handle.
Returns
SNSR_RC_OK for success, any other value indicates failure.
