Detailed Description

User-defined callback functions with associated data.

A SnsrCallback handle encapsulates a user-defined function with the SnsrHandler signature, arbitrary user data as a void *, and a clean-up function used to release the user data when the SnsrCallback handle is released.

SnsrCallback handles are used for all events generated by SnsrSession.

SNSR_API SnsrCallback snsrCallback ( SnsrHandler  h,
SnsrDataRelease  r,
void *  data 
)

Creates a new SnsrCallback instance.

These are used to invoke library callbacks and iterators.

  • New handles have a reference count of 0. Use snsrRetain() if keeping a reference and snsrRelease() to release such a reference.
  • SnsrHandler h will be called when this callback is invoked.
  • SnsrDataRelease r will be called when the callback reference count reaches 0 and is deleted. r must release the user data structure.
Parameters
[in]hthe function to call when this callback is invoked.
[in]ra function that releases data when the callback is destroyed. Use NULL if no clean-up is required.
[in]dataprivate user pointer.
Returns
A new callback instance. NULL if h is NULL or heap allocation failed.

snsrRun(), snsrForEach()
