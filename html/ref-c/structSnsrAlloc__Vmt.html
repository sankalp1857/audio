Detailed Description

SnsrAlloc virual method table.

This method table defines a heap allocator to use in place of malloc/realloc/free from stdlib.

The struct must remain valid until a call to snsrTearDown(), or the application ends.

// This minimal example uses the standard C library allocator.
static void *
vmtMalloc(void *ctx, size_t size) {
return malloc(size);
}
static void *
vmtRealloc(void *ctx, void *ptr, size_t size) {
return realloc(ptr, size);
}
static void
vmtFree(void *ctx, void *ptr)
{
free(ptr);
}
static const SnsrAlloc_Vmt CustomAlloc = {
vmtMalloc, vmtFree, vmtRealloc,
NULL, NULL, NULL, NULL, NULL, NULL, NULL
};
// In main, before any other snsr* calls
snsrConfig(SNSR_CONFIG_ALLOC, &CustomAlloc);
See also
snsrConfig, SNSR_CONFIG_ALLOC

SnsrAllocRC(* addPool) (void *ctx, void *pool, size_t size)

Add a new backing store pool to the allocator.

This method adds a new pool to the backing store used for the heap.

Note
This method is optional. If set to NULL, the allocator does not support adding pools to the heap store.
Parameters
[in]ctxThe value of the ctx member in this struct.
[in]poolvoid * to the start of a read-write memory segment to use as additional allocator store. This address must be aligned to the word size of the CPU.
[in]sizeThe number of bytes available at pool.
Returns
SNSR_ALLOC_RC_OK upon success, any other SnsrAllocRC on failure.

◆ malloc

void*(* malloc) (void *ctx, size_t size)

Allocate memory from the heap.

This method should work like the standard library's malloc() function. It should return a pointer to at least size bytes of writable memory, aligned to at least the word size of the CPU.

size will always be larger than 0.

Parameters
[in]ctxThe value of the ctx member in this struct.
[in]sizeNumber of bytes to allocate.
Returns
A pointer to an aligned segment of writable memory. NULL if memory could not be allocated.

◆ minPoolSize

size_t(* minPoolSize) (void *ctx, size_t maxAlloc, size_t maxCount)

Return an estimate of the pool size required.

This method returns an estimate of the smallest allocator pool required to repeat an instrumented run.

Note
This method is optional. If set to NULL, the estimated pool size will not be avialable in snsrAllocPerfStats().
Parameters
[in]ctxThe value of the ctx member in this struct.
[in]maxAllocThe high water mark of bytes allocated by the instrumented application.
[in]maxCountThe high water mark of the number of allocations made by the application.
Returns
An estimate of the smallest backing store pool that could satisfy maxAlloc and maxCount.

◆ size

size_t(* size) (void *ctx, void *ptr)

Return the size of the ptr allocation.

This method should work like malloc_size() or malloc_usable_size(). It should return the actual size of the block of memory allocated for ptr. This will typically be a bit larger than the requested size.

Note
This method is optional and used only by snsrAllocPerf(). If the allocation size is not available, set this method to NULL and avoid using snsrAllocPerf().
Parameters
[in]ctxThe value of the ctx member in this struct.
[in]ptrPointer to previously allocated heap segment.
Returns
The size of the memory block ptr points to.
