Conceptual Model

Overview of SDK concepts.

This library uses a dataflow approach to evaluate speech recognition tasks. It relies on inversion of control: User-defined hooks are invoked to report results and control task flow. Task models define flow pipelines and behavior. The API contains two primary data types: Session and Stream. Sessions encapsulate the entire state of a task instance, and all session input and output data are are vectored through streams.

Session instance behavior is customized via settings, of which there are three variants:

  • Configuration settings are serializable read-write values. These are used for parameter values and model objects. Task models contain reasonable default values - modify these only to tweak results.
  • Runtime settings are instance-specific and not serializable. These include callback handles, iterators, input streams and values such as the name of the phrase being enrolled.
  • Result settings are read-only values with very limited scope, such as the recognition results.

Stream instances are created on various underlying data streams, such as FILE * handles, memory segments, and live audio sources. Only the constructors are type-specific, all other functions are independent of the underlying stream type. Operations that take stream handles as arguments operate on any of the types.

Task evaluation typically follows this recipe:

  • Create a new session instance.
  • Load a task model into the instance.
  • Set the input source stream.
  • Register one or more event handlers.
  • Enter the main loop. The library will process the input streams and invoke event handlers at appropriate times. The main loop continues until a terminating condition is reached, such as an event returning an error code.
  • Release the session instance.

C phrase spotter

  • Handles returned by the library are reference-counted.
  • If an error occurs on a session handle all future operations on that handle will immediately return, until the error is explicitly cleared. This makes for more succinct error handling.
#include <stdio.h>
#include <snsr.h>
static SnsrRC
result(SnsrSession s, const char *key, void *privateData)
{
SnsrRC r;
const char *phrase;
double begin, end, score;
snsrGetString(s, SNSR_RES_TEXT, &phrase);
snsrGetDouble(s, SNSR_RES_BEGIN_MS, &begin);
snsrGetDouble(s, SNSR_RES_END_MS, &end);
r = snsrGetDouble(s, SNSR_RES_SV_SCORE, &score);
if (r != SNSR_RC_OK) return r;
printf("spotted: (%6.2lf) [%.0f, %.0f] ms %s\n", score, begin, end, phrase);
return SNSR_RC_STOP; /* Stop after a single spot */
}
int
main(int argc, char *argv[])
{
SnsrRC r;
SnsrSession s;
SnsrStream audio;
snsrNew(&s);
snsrLoad(s, snsrStreamFromFileName("spot-hbg-enUS-1.4.0-m.snsr", "r"));
snsrRequire(s, SNSR_TASK_TYPE, SNSR_PHRASESPOT); /* verify task type */
snsrSetStream(s, SNSR_SOURCE_AUDIO_PCM,
snsrStreamFromAudioDevice(SNSR_ST_AF_DEFAULT));
snsrSetHandler(s, SNSR_RESULT_EVENT, snsrCallback(result, NULL, NULL));
r = snsrRun(s);
if (r == SNSR_RC_STOP || r == SNSR_RC_STREAM_END) {
printf("Done. (%s)\n", snsrErrorDetail(s));
r = SNSR_RC_OK;
} else if(r != SNSR_RC_OK)
fprintf(stderr, "ERROR: %s\n", snsrErrorDetail(s));
snsrRelease(s);
return r;
}
