Detailed Description

This section describes the predefined stream type constructors.

SnsrStream wrappers are provided for <stdio.h> files, memory segments, FIFO buffers and audio recording devices. Additional types provide support for SnsrStream transformations, such as concatenation and format conversion.

Functions in this group that take SnsrStream arguments retain these handles and release them when they are no longer used. A SnsrStream handle with a zero reference count used as an argument will therefore be deallocated when it is no longer needed. Be sure to call snsrRetain() on handles that need to survive these function invocations.

SNSR_API SnsrStream snsrStreamFromAudioDevice ( SnsrStreamAudioFormat  format,
  ... 
)

Streams live audio from an audio capture device.

Creates a new SnsrStream attached to the specified audio device.

Note
Supports audio capture with ALSA on Linux, with Audio Queue Services on macOS and iOS, and with the Windows Multimedia Extentions Wave API on Windows.

SNSR_ST_AF_DEFAULT uses the default capture device with the format (16-bit little-endian LPCM) and sample rate (16 kHz) required by most SDK recognizers. This is the recommended format specifier.

Example:

// Use a specific ALSA device, rather than the default.
// The "plughw" device typically includes sample rate conversion,
// use this if the hardware does not support sampling at 16 kHz.
SnsrStream a = snsrStreamFromAudioDevice(SNSR_ST_AF_DEVICE, "plughw:1,0");
Parameters
[in]formatAudio device selector and format specifier.
Returns
A new SnsrStream handle or NULL if memory allocation failed.
See also
alsa-stream.c, wmme-stream.c
Examples
live-enroll.c, live-segment.c, live-spot.c, push-audio.c, and snsr-eval.c.

◆ snsrStreamFromAudioFile()

SNSR_API SnsrStream snsrStreamFromAudioFile ( const char *  filename,
const char *  mode,
SnsrStreamAudioFormat  format,
  ... 
)

Wraps a stream around an audio stdio file.

Creates a new SnsrStream using the <stdio.h> file functions.

This is equivalent to the code snippet below, with the difference that the RIFF header is updated after each write to the stream.

SnsrStream tmp;
tmp = snsrStreamFromFileName(argv[i], "w");
tmp = snsrStreamFromAudioStream(tmp, SNSR_ST_AF_DEFAULT);

Only a subset of the fopen() modes are supported:

  • r for read-only access.
  • w for write-only access, truncating the file to zero on open.

This stream type supports re-opening. The behavior follows that of fopen() with the given access mode.

Note
Requires FILE support in the standard C library.
Parameters
[in]filenamethe name of a file to open.
[in]moder or w.
[in]formatoutput target audio format.
Returns
A new SnsrStream handle or NULL if memory allocation failed.
Examples
live-enroll.c, push-audio.c, and snsr-eval.c.

◆ snsrStreamFromAudioStream()

SNSR_API SnsrStream snsrStreamFromAudioStream ( SnsrStream  a,
SnsrStreamAudioFormat  format,
  ... 
)

Converts stream audio format.

Creates a new SnsrStream on an existing stream, a. The new stream produces audio in format, converting as needed.

The audio format header in a will be read when the stream is first opened.

  • If this is a known, but unsupported format, snsrStreamOpen() will report an error.
  • If the format is not known, the data in a will be treated as a headerless little-endian 16-bit LPCM stream.

The initial implementation supports the 16-bit LPCM RIFF WAVE format, at 16 kHz sampling rate, for a single channel only.

Example:

SnsrStream a = snsrStreamFromFileName("hbg.wav", "r");
a = snsrStreamFromAudioStream(a, SNSR_ST_AF_DEFAULT);
snsrStreamOpen(a);
// read 16-bit PCM data from a
snsrRelease(a);
Note
When this stream type is used for output, all stream data will be buffered in memory until the stream is closed. The audio header and the data will then be written to a, the wrapped SnsrStream.
Parameters
[in]asource SnsrStream.
[in]formatoutput target audio format.
Returns
A new SnsrStream handle or NULL if memory allocation failed.
Examples
live-enroll.c, live-segment.c, and spot-enroll.c.

◆ snsrStreamFromBuffer()

SNSR_API SnsrStream snsrStreamFromBuffer ( size_t  initialSizeInBytes,
size_t  maxSizeInBytes 
)

Wraps a stream around a circular buffer.

Creates a new SnsrStream on a circular buffer with first-in-first-out semantics. The initial buffer will be large enough to hold initialSizeInBytes chars, and will dynamically grow up to maxSizeInBytes. If the SnsrStream the bufffer cannot be expanded, snsrStreamWrite() will set the SNSR_RC_EOF condition on the stream. This SnsrStream type does not support re-opening.

Note
snsrStreamRead() on streams of this type has non-blocking behavior, and will return SNSR_RC_EOF if the buffer runs empty.
Parameters
[in]initialSizeInBytesthe minimum size of the allocated ring buffer.
[in]maxSizeInBytesthe limit the buffer is allowed to expand to.
Returns
A new SnsrStream handle or NULL if memory allocation failed.
Examples
aqs-stream.c, push-audio.c, and wmme-stream.c.

SNSR_API SnsrStream snsrStreamFromCode ( SnsrCodeModel  identifier)

Wraps a stream around C code.

Creates a new SnsrStream from an opaque SnsrCodeModel handle.

The SnsrCodeModel handle is an identifier loaded from an object file, compiled from C code generated by a call to snsrSave() with a SNSR_FM_SOURCE format argument. These data are typically mapped to the read-only TEXT (code) segment.

Use this on platforms where the TEXT segment is read from ROM to reduce the amount of RAM a model needs to run.

Example:

extern SnsrCodeModel snsrmodel;
SnsrSession s;
snsrNew(&s);
snsrLoad(s, snsrStreamFromCode(snsrmodel));

[in]identifierSnsrCodeModel handle.

A new SnsrStream handle or NULL if memory allocation failed.
See also
snsrSave(), SNSR_FM_SOURCE

◆ snsrStreamFromFileName()

SNSR_API SnsrStream snsrStreamFromFileName ( const char *  filename,
const char *  mode 
)

Wraps a stream around a stdio file.

Creates a new SnsrStream using the <stdio.h> file functions.

Only a subset of the fopen() modes are supported:

  • r for read-only access.
  • w for write-only access, truncating the file to zero on open.
  • a for write-only append access.

mode supports an optional second character:

  • b for binary mode. This is the default if mode contains just one character.
  • t for text mode. On Windows this translates line-endings between \n and \r\n.

This stream type supports re-opening. The behavior follows that of fopen() with the given access mode.

Note
Requires FILE support in the standard C library.
Parameters
[in]filenamethe name of a file to open.
[in]moder, w, or a, followed by optional b or t.
Returns
A new SnsrStream handle or NULL if memory allocation failed.
Examples
live-enroll.c, live-segment.c, live-spot-stream.c, live-spot.c, push-audio.c, snsr-edit.c, snsr-eval.c, spot-convert.c, and spot-enroll.c.

◆ snsrStreamFromOpenStream()

SNSR_API SnsrStream snsrStreamFromOpenStream ( SnsrStream  source,
size_t  sizeInBytes 
)

Wraps a stream around an open stream.

Creates a new SnsrStream on an existing, open SnsrStream. The new stream mode (read or write) matches the source stream. Read and write operations are passed through to the source stream, but the number of bytes read or written is limited to sizeInBytes. In- and output operations will set the stream status to SNSR_RC_EOF when this limit is reached.

This function will retain source, and release it when the stream is closed. Be sure to snsrRetain() the source handle if it is to remain valid.

Closing this SnsrStream will not close source.

Parameters
[in]sourcesource stream.
[in]sizeInBytesnew stream read or write limit.
Returns
A new SnsrStream handle or NULL if memory allocation failed.

◆ snsrStreamFromStreams()

SNSR_API SnsrStream snsrStreamFromStreams ( SnsrStream  a,
SnsrStream  b 
)

Concatenates two streams.

Creates a new SnsrStream that is a concatenation of a and b.

Streams a and b will be each be opened exactly once. These should be unopened (snsrStreamGetMeta() for SNSR_ST_META_OPEN_COUNT must be 0) when snsrStreamFromStreams() is called. This limitation does not apply to streams returned by this function, these can be nested to arbitrary depth to concatenate more than two streams.

The new stream will read from (or write to) a until end-of-stream is reached, then switch to stream b until that is exhausted, at which point SNSR_RC_EOF is returned.

Streams a and b must have identical modes. If one stream was opened for reading, and the other for writing, this function will set the error code to SNSR_RC_WRONG_MODE.

Closing this stream will close the current source stream. The next open call will open the next source stream. When no further source streams remain snsrStreamOpen() returns SNSR_RC_CANNOT_REOPEN.

Example:

SnsrStream s;
s = snsrStreamFromStreams(
snsrStreamFromFileName("one.txt", "w"),
snsrStreamFromStreams(snsrStreamFromFileName("two.txt", "w"),
snsrStreamFromFILE(stdout, SNSR_ST_MODE_WRITE)));
snsrStreamOpen(s);
snsrStreamPrint(s, "This writes to \"one.txt\"\n");
snsrStreamClose(s);
snsrStreamOpen(s);
snsrStreamPrint(s, "This writes to \"two.txt\"\n");
snsrStreamClose(s);
snsrStreamOpen(s);
snsrStreamPrint(s, "This writes to stdout\n");
snsrRelease(s);
Parameters
[in]afirst source stream.
[in]bsecond source stream.
Returns
A new SnsrStream handle or NULL if memory allocation failed.
Examples
live-enroll.c, and snsr-eval.c.

◆ snsrStreamRaise()

SNSR_API SnsrStream snsrStreamRaise ( SnsrStreamEvent  e,
SnsrDataRelease  r,
void *  data 
)

Inserts an event marker into a stream.

Creates a new SnsrStream that is readable, but provides no data. Any read or skip operation will immediately return end-of-stream.

Calls e(b, data) when SnsrStream b is opened, unless e == NULL.

Calls r(data) when b is released, unless r == NULL. This function should release the data handle.

Use this stream type with snsrStreamFromStreams() to embed events in a stream.

Parameters
[in]ethe function to call when this stream is opened. Must not be NULL.
[in]ra function that releases data when this stream is deallocated. Use NULL if no clean-up is required.
[in,out]datauser pointer, the data parameter of SnsrStreamEvent.
Returns
A new SnsrStream handle or NULL if memory allocation failed.
