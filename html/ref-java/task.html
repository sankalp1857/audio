Phrase Spotter

Raises an event when a preconfigured phrase is spotted in the input audio stream. Multiple phrases can be active at the same time. The Spotter Enrollment task provides a convenient way to enroll user-defined phrases, which can also be used for speaker verification.

Behavior

  • Reads audio data from the input stream Snsr.SOURCE_AUDIO_PCM.
  • Invokes Snsr.SAMPLES_EVENT.
  • If one of the phrases in the model is spotted, invokes Snsr.RESULT_EVENT.
  • Continues processing until SnsrRC.EOF is encountered on the input stream, or one of the event handlers return a code other than SnsrRC.OK.

Spotter Enrollment

Fixed and User-Defined Trigger enrollment details.

Behavior

Interactive mode (the default)

Snsr.INTERACTIVE_MODE == 1

  • Invokes Snsr.RESUME_EVENT.
  • Reads audio data from the input stream Snsr.SOURCE_AUDIO_PCM.
  • Invokes Snsr.SAMPLES_EVENT.
  • Repeats until an enrollment segment is detected.
  • If Snsr.USER is not set, invokes Snsr.NEXT_EVENT.
  • Invokes Snsr.PAUSE_EVENT.
  • Checks enrollment recording, invokes Snsr.PASS_EVENT if good, Snsr.FAIL_EVENT if not.
  • Repeats until the required number of enrollments have been recorded.
  • Validates all enrollments, invokes Snsr.PASS_EVENT if good, Snsr.FAIL_EVENT if not.
  • Invokes Snsr.NEXT_EVENT. If a new user is encountered repeats from the top.
  • Invokes Snsr.ENROLLED_EVENT.
  • Starts adaptation, invoking Snsr.PROG_EVENT periodically.
  • Invokes Snsr.DONE_EVENT.

Batch mode

Snsr.INTERACTIVE_MODE == 0

  • Invokes Snsr.RESUME_EVENT.
  • Reads audio data from the input stream Snsr.SOURCE_AUDIO_PCM.
  • Invokes Snsr.SAMPLES_EVENT.
  • Repeats until an enrollment segment is detected.
  • If Snsr.USER is not set, invokes Snsr.NEXT_EVENT.
  • Invokes Snsr.PAUSE_EVENT.
  • Checks enrollment recording, invokes Snsr.PASS_EVENT if good, Snsr.FAIL_EVENT if not.
  • Invokes Snsr.RESUME_EVENT.
  • Repeats from the top until Snsr.SOURCE_AUDIO_PCM reaches SnsrRC.EOF.
  • Validates all enrollments, invokes Snsr.PASS_EVENT if good, Snsr.FAIL_EVENT if not.
  • Repeats until Snsr.USER is set to NULL.
  • Invokes Snsr.ENROLLED_EVENT.
  • Starts adaptation, invoking Snsr.PROG_EVENT periodically.
  • Invokes Snsr.DONE_EVENT.

Example

  • See the EnrollTrigger example app in the Quick Start section.

Voice Activity Detector

Segments a single speech-like region from an audio stream.

Behavior

  • Reads audio data from the input stream Snsr.SOURCE_AUDIO_PCM.
  • Invokes Snsr.SAMPLES_EVENT.
  • Invokes Snsr.BEGIN_EVENT when speech is first detected.
  • Invokes an endpoint event (Snsr.END_EVENT, Snsr.LIMIT_EVENT, or Snsr.SILENCE_EVENT) when the final speech point has been determined.
  • Continues processing until SnsrRC.EOF is encountered on the input stream, or one of the event handlers return a code other than SnsrRC.OK.
    • The default behavior for endpoint events is to return SnsrRC.STOP.
  • Between the Snsr.BEGIN_EVENT and the endpoint event audio samples from Snsr.SOURCE_AUDIO_PCM are copied to Snsr.SINK_AUDIO_PCM.
    • See Snsr.INCLUDE_LEADING_SILENCE, Snsr.PASS_THROUGH.
  • Endpoint events restart the VAD using the same settings.

Example

and segmentSpottedAudio.java

Settings

  • Task type validation
    • snsrRequire(session, SNSR_TASK_TYPE, SNSR_VAD)
    • Snsr.TASK_VERSION == 1.0.0.
  • Configuration settings
    • Snsr.BACKOFF, Snsr.HOLD_OVER, Snsr.INCLUDE_LEADING_SILENCE, Snsr.LEADING_SILENCE, Snsr.MAX_RECORDING, Snsr.PASS_THROUGH, Snsr.TRAILING_SILENCE, Snsr.VAD.
  • Runtime settings
    • Snsr.SKIP_TO_MS, Snsr.SKIP_TO_SAMPLE.
    • Handlers
      • The runtime and result settings below are available only during the execution of the handler functions.
      • Snsr.BEGIN_EVENT
        • Results: Snsr.RES_BEGIN_MS, Snsr.RES_BEGIN_SAMPLE.
      • Snsr.END_EVENT, Snsr.LIMIT_EVENT
        • Results: Snsr.RES_BEGIN_MS, Snsr.RES_BEGIN_SAMPLE, Snsr.RES_END_MS, Snsr.RES_END_SAMPLE.
      • Snsr.SAMPLES_EVENT
        • Result: Snsr.RES_SAMPLES.
      • Snsr.SILENCE_EVENT
        • No speech detected: No segmentation results available.
    • Streams
      • Snsr.SOURCE_AUDIO_PCM, Snsr.SINK_AUDIO_PCM.

Phrase Spotter with VAD

This task is a combination of Phrase Spotter and Voice Activity Detector. The audio directly following the end of the spotted phrase is gated by the VAD, which provides start- and end-pointing.

Use this task type for spotter-to-search applications, where the VAD audio is sent to a third party recognition engine.

Behavior

  • Reads audio data from the input stream Snsr.SOURCE_AUDIO_PCM.
  • Invokes Snsr.SAMPLES_EVENT.
  • If one of the phrases in the model is spotted:
    • Invokes Snsr.RESULT_EVENT.
    • Invokes Snsr.LISTEN_BEGIN_EVENT.
    • Processes audio following the phrase with the VAD.
      • Invokes Snsr.BEGIN_EVENT when speech is first detected.
      • Invokes an endpoint event (Snsr.END_EVENT, Snsr.LIMIT_EVENT, or Snsr.SILENCE_EVENT) when the final speech point has been determined.
      • Between the Snsr.BEGIN_EVENT and the endpoint event audio samples from Snsr.SOURCE_AUDIO_PCM are copied to Snsr.SINK_AUDIO_PCM.
      • See Snsr.INCLUDE_LEADING_SILENCE, Snsr.PASS_THROUGH.
    • Invokes Snsr.LISTEN_END_EVENT and switches audio processing back to the phrase spotter.
  • Continues processing until SnsrRC.EOF is encountered on the input stream, or one of the event handlers return a code other than SnsrRC.OK.

Example

With a phrase spotter for "Alexa", the phrase "Alexa will it rain in Portland tomorrow?" will provide the audio for "will it rain in Portland tomorrow?" in the Snsr.SINK_AUDIO_PCM stream. The application can register handlers to receive Snsr.BEGIN_EVENT and Snsr.END_EVENT callbacks.

Settings

This task includes all settings from Phrase Spotter and Voice Activity Detector.

  • Additional runtime settings
    • Snsr.SINK_AUDIO_PCM contains the audio samples selected by the VAD.

Large Vocabulary Continuous Speech Recognizer

Converts speech to text. Raises events when partial and final hypotheses are available.

Task version 1.1.0 and later includes support for a lightweight experimental VAD. This uses the <s> and </s> delimiters at the start and end of the recoginition results, and does not consider the speech energy levels.

Behavior

  • Reads audio data from the input stream Snsr.SOURCE_AUDIO_PCM.
  • Invokes Snsr.SAMPLES_EVENT.
  • Invokes Snsr.PARTIAL_RESULT_EVENT with the current best hypothesis once every Snsr.PARTIAL_RESULT_INTERVAL ms. This is disabled when Snsr.PARTIAL_RESULT_INTERVAL is set to 0.
  • When SnsrRC.EOF is encountered in the input stream, or when a Voice Activity Detector endpoint is detected:
    • If any NLU semantic tags are included in the word-level results, invokes Snsr.NLU_SLOT_EVENT for each top-level tag encountered.
    • Invokes Snsr.RESULT_EVENT with the final hypothesis.
  • Continues processing until SnsrRC.EOF is encountered on the input stream, or one of the event handlers return a code other than SnsrRC.OK.

Settings

  • Task type validation
    • snsrRequire(session, SNSR_TASK_TYPE, SNSR_LVCSR)
    • Snsr.TASK_VERSION == 1.1.0.
  • Configuration settings
    • Snsr.LVCSR, Snsr.OOV_REJECT, Snsr.PARTIAL_RESULT_INTERVAL, Snsr.SHOW_SILENCE, Snsr.SAMPLE_RATE, Snsr.AUDIO_STREAM_SIZE, Snsr.AUDIO_STREAM_FROM, Snsr.AUDIO_STREAM_TO.
    • Snsr.BACKOFF, Snsr.HOLD_OVER, Snsr.LEADING_SILENCE, Snsr.MAX_RECORDING, Snsr.TRAILING_SILENCE.
  • Runtime settings
    • Handlers
      • The runtime and result settings below are available only during the execution of the handler functions.
      • Snsr.BEGIN_EVENT
        • Results: Snsr.RES_BEGIN_MS, Snsr.RES_BEGIN_SAMPLE.
      • Snsr.END_EVENT, Snsr.LIMIT_EVENT
        • Results: Snsr.RES_BEGIN_MS, Snsr.RES_BEGIN_SAMPLE, Snsr.RES_END_MS, Snsr.RES_END_SAMPLE.
      • Snsr.SILENCE_EVENT
        • No speech detected: No segmentation results available.
      • Snsr.NLU_SLOT_EVENT
        • Iterators: Snsr.NLU_SLOT_LIST.
        • Results: Snsr.RES_NLU_SLOT_COUNT, Snsr.RES_NLU_SLOT_NAME, Snsr.RES_NLU_SLOT_VALUE.
      • Snsr.PARTIAL_RESULT_EVENT
        • Iterators: Snsr.PHRASE_LIST, Snsr.WORD_LIST.

      • Snsr.RESULT_EVENT
        • Iterators: Snsr.PHRASE_LIST, Snsr.WORD_LIST.
        • Results: Snsr.RES_BEGIN_MS, Snsr.RES_BEGIN_SAMPLE, Snsr.RES_END_MS, Snsr.RES_END_SAMPLE, Snsr.RES_TEXT.
      • Snsr.SAMPLES_EVENT
        • Result: Snsr.RES_SAMPLES.
    • Iterators
      • Snsr.NLU_SLOT_LIST.
        • Results: Snsr.RES_NLU_SLOT_COUNT, Snsr.RES_NLU_SLOT_NAME, Snsr.RES_NLU_SLOT_VALUE.
      • Snsr.PHRASE_LIST
        • Results: Snsr.RES_BEGIN_MS, Snsr.RES_BEGIN_SAMPLE, Snsr.RES_END_MS, Snsr.RES_END_SAMPLE, Snsr.RES_TEXT.
      • Snsr.WORD_LIST
        • Results: Snsr.RES_BEGIN_MS, Snsr.RES_BEGIN_SAMPLE, Snsr.RES_END_MS, Snsr.RES_END_SAMPLE, Snsr.RES_TEXT.

        • Results: Snsr.RES_BEGIN_MS, Snsr.RES_BEGIN_SAMPLE, Snsr.RES_END_MS, Snsr.RES_END_SAMPLE, Snsr.RES_TEXT.
    • Streams

Task Templates

Task templates are models with custom behavior, and one or more empty slots that can be filled with phrase spotter modules. Slots can be filled during development using the snsr-edit tool, or at runtime with the SnsrSession.setStream() API call.

This release includes these templates:

tpl-spot-concurrent

  • Runs the spotters in Snsr.SLOT_0 and Snsr.SLOT_1 at the same time, in the same thread.
  • The combined model is a Phrase Spotter, and can be used in any application without API changes.
    • Most combined model settings affect both embedded spotters. Settings Snsr.OPERATING_POINT, Snsr.OPERATING_POINT_LIST, and Snsr.RES_AVAILABLE_POINT are not available in this template.
    • Change individual settings at runtime by prefixing the setting name with the slot:
      /* Set the operating point for spotter 0 only. */
      snsrSetInt(session, SNSR_SLOT_0 SNSR_OPERATING_POINT, 7);
  • Note that the two phrase spotters are independent, and can produce results that overlap in time. For production use Sensory recommends custom multi-phrase phrase spotters instead. These have improved false reject / false accept performance.

tpl-spot-debug

  • Use this for spotter debug logging.
  • Runs the spotter in Snsr.SLOT_0.
  • Requires a path to the output logfile in Snsr.DEBUG_LOG_FILE.
  • Snsr.INCLUDE_MODEL controls whether the log file includes a copy of the original task model. This is enabled by default.
  • The combined model is a Phrase Spotter, and can be used in any application without API changes.
  • Logs record:
    • SDK version and acceleration in play,
    • the spotter model being used,
    • audio, and
    • event callbacks.
  • Extract text, model and audio data from the log file with the snsr-log-split utility, see snsr-log-split.

tpl-spot-select

  • Use Snsr.SLOT to select either the spotter in Snsr.SLOT_0 or in Snsr.SLOT_1.
    • ‡ Note that tpl-spot-vad-lvcsr an tpl-vad-lvcsr conform to the Phrase Spotter API, and can be used in either slot in this template.
  • Use this template to reduce the model size when an application uses variants of the same recognizer in different contexts. If, for example, an application uses both tpl-spot-vad-lvcsr and tpl-vad-lvcsr with the same LVCSR recognizer, slotting these into tpl-spot-select will roughly halve the total size required. The reason being that model files contain just one copy of each unique object.
  • The template model is a Phrase Spotter, and can be used in any application that supports this type without API changes.

tpl-spot-sequential

  • With Snsr.LOOP set to 0 (the default): Runs the spotter in Snsr.SLOT_0 until it spots, then runs Snsr.SLOT_1 until it spots, or the Snsr.LISTEN_WINDOW timeout expires, then returns to the spotter in Snsr.SLOT_0.
  • With Snsr.LOOP set to 1: Runs the spotter in Snsr.SLOT_0 until it spots, then runs Snsr.SLOT_1 until the Snsr.LISTEN_WINDOW timeout expires, then returns to the spotter in Snsr.SLOT_0. Resets the expiration timer every time Snsr.SLOT_1 spots.
  • Use this to listen for a trigger phrase which is followed by a command. For example, "Hello blue genie, play music."
  • The combined model is a Phrase Spotter, and can be used in any application without API changes.
    • Combined model settings refer to the spotter in Snsr.SLOT_1.
    • Change settings for Snsr.SLOT_0 by prefixing the setting name with Snsr.SLOT_0.
  • The Snsr.LISTEN_BEGIN_EVENT is raised just before audio focus switches to Snsr.SLOT_1, and Snsr.LISTEN_END_EVENT fires just before audio focus switches back to Snsr.SLOT_0. If there's no Snsr.RESULT_EVENT between these events it is an indication that the Snsr.LISTEN_WINDOW on Snsr.SLOT_1 has timed out.

tpl-spot-vad

  • Runs the spotter in Snsr.SLOT_0 until it spots, then does start- and endpoint detection on the subsequent audio stream using a Voice Activity Detector.
  • Use this for trigger-to-search applications.
  • The combined model is a Phrase Spotter with VAD.
  • Set Snsr.INCLUDE_LEADING_SILENCE to 1 to include the trigger audio in the VAD audio output stream.

tpl-spot-vad-lvcsr

  • Runs the spotter in Snsr.SLOT_0 until it spots, does start- and endpoint detection on the subsequent audio stream using a Voice Activity Detector, and sends this segmented audio to the large-vocabulary recognizer in Snsr.SLOT_1.
  • Snsr.PARTIAL_RESULT_EVENT and Snsr.RESULT_EVENT refer to the large-vocabulary recognizer in Snsr.SLOT_1. If you need direct access to spotter results, prefix these settings with Snsr.SLOT_0.
  • The combined model is a Phrase Spotter.
  • Set Snsr.INCLUDE_LEADING_SILENCE to 1 to pass the trigger audio to the LVCSR recognizer.

tpl-vad-lvcsr

  • Does start- and endpoint detection on Snsr.SOURCE_AUDIO_PCM, using a Voice Activity Detector, then sends this segmented audio to the large-vocabulary recognizer in Snsr.SLOT_0.
  • The combined model is a Phrase Spotter.
