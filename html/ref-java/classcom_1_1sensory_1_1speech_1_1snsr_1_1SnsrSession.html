Detailed Description

Primary SDK runtime class.

A SnsrSession handle contains the entire state for a task.

Instance method calls are not re-entrant. If these are called from different threads, all such calls must be protected by application-level synchronization.

Methods that are not explicitly used to query the state return the instance itself, to support method chaining. For example:

SnsrSession s = new SnsrSession()
.load("task-data.snsr")
.require(Snsr.TASK_TYPE, Snsr.PHRASESPOT);

◆ dup()

SnsrSession dup ( )

Duplicates a SnsrSession instance.

Creates a new SnsrSession instance that is a clone of the source instance. This function is roughly equivalent to:

SnsrStream buf = SnsrStream.fromBuffer(1024, 1073741824);
a.save(SnsrDataFormat.CONFIG, buf);
SnsrSession b = new SnsrSession().load(buf);
buf.delete();
buf = null;

The new instance contains all of the configuration from the source, but none of the runtime settings. Immutable configuration settings (such as acoustic models) are shared between the source and the new instance. Using this method to create two instances of a task will therefore use less memory than loading the same task data into two SnsrSession instances.

Returns
A duplicate SnsrSession instance.

References SnsrSession.SnsrSession().

◆ push()

SnsrSession push ( String  name,
byte []  data 
)

Processes data in a session.

Adds data to the name input SnsrStream for SnsrSession s, then process these and other buffered stream data. Invokes SnsrSession.Listener registered in the session and (if defined) writes to output streams.

Processing will continue until:

  • All data are ingested.
  • Or one of the invoked callbacks returns an error code, which SnsrSession.push() will return. SnsrRC.INTERRUPTED, SnsrRC.REPEAT, SnsrRC.SKIP, SnsrRC.STOP or SnsrRC.TIMED_OUT should be used to indicate that the operation was stopped on request.
  • Or the Snsr.DURATION_LIMIT timeout is reached.

SnsrSession.Listener are invoked on the thread that SnsrSession.push() was called on. The processing loop is single-threaded, so it stalls while waiting for user functions to return.

If Snsr.DURATION_LIMIT is used, SnsrSession.push() has to defer processing when the duration limit is reached. In this case, data are added to an internal ring buffer. SnsrSession.push() will report SnsrRC.NOT_ENOUGH_SPACE if no ring ring buffer space remains for this deferral. To address such a failure, you could:

  • Increase the Snsr.PUSH_BUFFER_SIZE
  • Increase the Snsr.PUSH_DURATION_LIMIT
  • Use a model with lower CPU requirements.
Since
TrulyNatural SDK 6.20.
Parameters
namethe name of an input stream, such as Snsr.SOURCE_AUDIO_PCM.

◆ require()

SnsrSession require ( String  key,
String  value 
)

Validates task requirements.

Checks that the value of the key matches the required value value.

If key is Snsr.TASK_VERSION, this function uses semantic version precedence to determine whether the task version meets the value version requirement.

value is a semantic version range:

  • A range is composed of one or more sets, joined by " || ". A version matches a range if, and only if, every comparator in at least one of the " || " -separated sets is satisfied by the version.
  • A set is a list of comparators separated by whitespace. A set is satisfied by the intersection of all the comparators it includes.
  • A comparator is an operator followed by a version.
  • operators include:
    • =, <, <=, >, >=: range comparisons.
    • ~: taskVersion >= value && taskVersion < M where M is the next major version number.
    • ^: taskVersion >= value && taskVersion < M where M is the next major version number, skipping over zeros from the left. For taskVersion >= 1.0.0 this is equivalent to the ~ operator. ^0.minor.patch allows only patch updates, e.g. ^0.1.1 will accept 0.1.2 but not 0.2.0.
    • The default behavior when no operator is specified is ^.

This method is used as a sanity check, to verify that the task model loaded is of the correct type, and that it has a compatible version. Example:

session.require(Snsr.TASK_VERSION, "0.5.0 || 1.0.0");
Parameters
keythe name of a setting to validate, such as Snsr.TASK_TYPE or Snsr.TASK_VERSION.
valuestring to match the key setting against, such as Snsr.PHRASESPOT.
Returns
The SnsrSession instance.

◆ reset()

SnsrSession reset ( )

Resets a session to its initial state.

This call discards all pending results, clears internal buffers, and resets session time and sample counts to 0.

This method is most useful to clear the state of a template model (such as tpl-spot-vad) that was stopped through a callback.

Note
Call reset() only after the run() method has returned. Invoking reset() in a callback handler will lead to undefined behavior.

◆ run()

SnsrSession run ( ) throws java.io.IOException

Processes stream data in a session.

Enters the main SnsrSession processing loop. This function will process data read from input streams, invoke SnsrSession.Listener registered in the session and (if defined) write to output streams.

Processing will continue until:

  • One of the data stream reach SnsrRC.EOF, or
  • one of the invoked callbacks return an error code.

Return codes other than SnsrRC.OK, SnsrRC.STREAM_END, SnsrRC.STOP, SnsrRC.SKIP, SnsrRC.REPEAT, and SnsrRC.TIMED_OUT will raise an exception.

Returns
The SnsrSession instance.
Exceptions

Serializes a session to a stream.

Saves the configuration or runtime settings of the session instance to an output stream.

SnsrDataFormat specifies what is serialized:

  • SnsrDataFormat.CONFIG serializes the task and all of its configuration settings.
  • SnsrDataFormat.RUNTIME serializes the runtime settings only, such as phrase spotter enrollments in Snsr.ENROLL tasks.
  • SnsrDataFormat.SOURCE serializes the task and all of its configuration settings as compilable C code. Use this on platforms where code is directly read from ROM to reduce RAM requirements. Note that loading the produced code requires use of the C TrulyNatural API.

◆ stop()

SnsrSession stop ( )

Stops session processing.

When used with pull mode audio processing, this method will cause SnsrSession.run() to stop and return SnsrRC.STOP. It is safe to call SnsrSession.stop() from a different processing thread.

When used with push mode audio processing and SnsrSession.push(), this method flushes the session processing pipeline, then stops any internal processing threads started for the session.

Note
Do not call SnsrSession.stop() on the SnsrSession handle in an event callback, return SnsrRC.STOP from the callback instead.
Since
TrulyNatural SDK 6.20.
Returns
The SnsrSession instance.
