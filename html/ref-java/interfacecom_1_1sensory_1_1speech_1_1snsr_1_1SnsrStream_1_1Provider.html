Detailed Description

Provider API for user-defined stream types.

Use this interface to add new SnsrStream types.

Each SnsrStream type exposes a single API function used to create a new instance of this type. The constructor function sets up the required data structures and copies arguments into these structures for use by the Provider.onOpen() method.

The constructor must not open the underlying data stream. Provider.onOpen() will do this instead. Delaying the open operation allos for function compositions, such as a SnsrStream that is a concatenation of other streams.

Use SnsrStream.fromProvider() to create a SnsrStream instance from a SnsrStream.Provider instance.

◆ onClose()

long onClose ( ) throws IOException

Closes the underlying data stream.

  • Called in response to a SnsrStream.close() API call.
  • Will only be called on open SnsrStream handles.
  • Buffered output must be flushed.
  • Buffered input must be discarded.
  • Must return Provider.OK if the operation succeeded, and an error code if the underlying resource could not be closed.

◆ onOpen()

long onOpen ( ) throws IOException

Opens the underlying data stream.

  • Called in response to a SnsrStream.open() API call.
  • Will only be called on closed SnsrStream handles.
  • Must open the underlying stream in binary mode, unless documented otherwise.
  • Must return Provider.OK if the operation succeeded, and an error code (Provider.NOT_OPEN) if the resource could not be opened.

◆ onRead()

long onRead ( byte []  buffer) throws IOException

Reads data and makes it available as a SnsrStream.

  • Called in response to a SnsrStream.read() API call.
  • The handle will be open when this function is called.
  • Must read buffer.length bytes into buffer.
  • The read must block (i.e. not return) until all the requested data have been read, or an error is encountered.
  • Must return the number of actual bytes read, unless an error condition was encountered.
    • Returning fewer than buffer.length bytes indicates that the end of the stream was reached.
  • Must return Provider.INTERRUPTED if the read was interrupted.

◆ onRelease()

void onRelease ( )

Releases private SnsrStream data.

  • Called just before the SnsrStream handle is de-allocated.
  • The handle will have been closed when this function is called - there is no need to check.
  • Must release resources allocated by the type's constructor.

◆ onWrite()

long onWrite ( byte []  buffer) throws IOException

Writes data to the underlying data stream.

  • Called in response to a SnsrStream.write() API call.
  • The handle will be open when this function is called.
  • Must write buffer.length bytes from buffer.
  • The write must block (i.e. not return) until all of the requested data have been written to the underlying stream, or an error is encountered.
  • Must return the number of bytes actually written, unless an error occurred.
  • Return Provider.INTERRUPTED if the write was interrupted.
