Task Models

The model/ subdirectory contains sample task model data files.

Use a groupId of com.sensory.speech.snsr.model when referring to these models in a Gradle build script dependency list, for example:

snsr_model "com.sensory.speech.snsr.model:spot-hbg-enUS:1.4.0:m@snsr"

Models marked with ‡ are exclusive to the TrulyNatural SDK.

Example task models:

  • Phrase Spotters
  • Spotter Enrollment
  • Voice Activity Detector
  • ‡ Large Vocabulary Continuous Speech Recognizer
  • Templates

â€¡ Large Vocabulary Continuous Speech Recognizer

In general, resource-limited devices do not have the compute power to allow for a low error rate when the task is unbounded. Even if it was able to, it is not clear if the device would be able to do a useful task or know how to react on any random speech input. The device will typically be tasked to do something specific in a known context, which limits the likelihood of out of context speech significantly, and while the user could still say anything, it is usually sufficient to just ignore such input. For such tasks, we care about detecting when some speech input is out-of-domain, but we do not really care if we mis-recognize the exact words. This allows us to use a small and lightweight (in terms of compute resources) background model to catch out-of-domain phrases, and target our limited compute resources to accurately recognize in-domain phrases. These are the background models we ship.

In some cases there may be sufficient compute resources to target a much larger domain and for such problems we include a large broad-domain model that is targeted to a range of problems that are broadly speaking in the command-and-control domain. This could be used as a started model that can be be better targeted to an actual use case by collecting data, merging that with the broader model and as actual domain data increase, decrease the "fill-in" data from the broader model.

See Large Vocabulary Continuous Speech Recognizer for a list of settings these models support.

‡ lvcsr-background-enUS-1.2.1.snsr

US English model with support for Custom Recognition, includes a small (< 1 MiB) background language model to recognize speech input that is not part of the custom grammar.

Note
The background recognizer is optimized for speed and size. It has a small vocabulary and will mis-recognize most words. Use it only to detect utterances not covered by the ~custom grammar you provide.

The top-level grammar in this model is:

g = ~spotter ({background ~background} | ~custom);

Default values:

  • ~spotter contains a class built from grammar = :;, it doesn't consume any input or produce any output.
  • ~custom is not defined.
  • ~background contains a small recognizer optimized to recognize out-of-domain speech input.
$ snsr-eval -t model/lvcsr-background-enUS-1.2.1.snsr \
-s partial-result-interval=0 \
-f grammar-stream.custom data/grammars/enrollments-nlu-slot.txt \ data/enrollments/jackalope-1-0-c.wav
NLU intent: navigate = i'm running low on gas
NLU entity: place = gas
375 2265 jackalope i'm running low on gas

‡ stt-enUS-automotive-medium-2.3.13.snsr

Speech-To-Text English recognizer with broad-domain support and special focus on automotive command-and-control tasks. It includes a machine-learned NLU component that identifies automotive intents and entities.

This model requires STT support, which currently depends on third-party Open Source modules that are optionally included in the TrulyNatural SDK. See Open Source Licenses for details.

This model does not support Custom Recognition or lightweight VAD.

All model components (acoustic, language, and NLU) are owned by Sensory.

tpl-spot-sequential-1.5.0.snsr

Runs the spotter model in Snsr.SLOT_0 until it spots, then the model in Snsr.SLOT_1.

Use this to create a recognizer for a wake word followed by one (or more if Snsr.LOOP == 1) commands from a set of phrases.

See tpl-spot-sequential.

tpl-spot-vad-3.8.0.snsr

Runs Voice Activity Detector on audio following a phrase spotted by the model in Snsr.SLOT_0. Suitable for spot-to-search applications. Uses a deep-learned VAD for improved segmentation accuracy.

See tpl-spot-vad.

‡ tpl-spot-lvcsr-1.5.0.snsr

Note
Removed from version 7.2.0 and later. Use tpl-spot-vad-lvcsr-3.15.0.snsr instead.

Runs spotter model in Snsr.SLOT_0 until it spots, then the LVCSR recognizer in Snsr.SLOT_1. Requires the (experimental) built-in VAD support in the LVCSR recognizer.

Example:

$ snsr-eval -v -t model/tpl-spot-lvcsr-1.5.0.snsr \
-f 0 model/spot-voicegenie-enUS-6.5.1-m.snsr \
-f 1 model/lvcsr-background-enUS-1.2.1.snsr \
-s trailing-silence=1000
Note
You must explicitly set Snsr.TRAILING_SILENCE when using this template. If the trailing silence time-out is not set, the template will not invoke Snsr.RESULT_EVENT, and will not return to listening for the spotter phrase.

See tpl-spot-vad.

‡ tpl-spot-vad-lvcsr-3.15.0.snsr

Runs spotter model in Snsr.SLOT_0 until it spots, then the LVCSR recognizer in Snsr.SLOT_1, gated by a Voice Activity Detector. Uses a deep-learned VAD for improved segmentation accuracy.

See tpl-spot-vad-lvcsr.

‡ tpl-vad-lvcsr-3.14.0.snsr

Runs the LVCSR recognizer in Snsr.SLOT_0 gated by a deep-learned Voice Activity Detector.

See tpl-vad-lvcsr.

‡ tpl-opt-spot-vad-lvcsr-1.20.0.snsr

Runs a large vocabulary recognizer gated by either a spotted phrase, or just a Voice Activity Detector.

The template has two slots, Snsr.SLOT_0 (or Snsr.PHRASESPOT as an alias) for the spotter model, and Snsr.SLOT_1 (or Snsr.LVCSR) for the large vocabulary recognizer.

Set Snsr.SLOT to Snsr.SLOT_0 to gate the recognizer with the spotted phrase, and set Snsr.SLOT to Snsr.SLOT_1 to gate it with the VAD instead. If Snsr.SLOT isn't set, it defaults to Snsr.SLOT_0.

Example use:

% snsr-edit -vv -o spot-or-vad-lvcsr.snsr\
-t model/tpl-opt-spot-vad-lvcsr-1.20.0.snsr\
-f phrasespot model/spot-voicegenie-enUS-6.5.1-m.snsr\
