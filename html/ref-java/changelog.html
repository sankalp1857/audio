On a near-miss false reject, these models switch to a more accepting operating point for a limited duration to make it more likely that the model will accept a repeated wake word utterance. Enable this feature (on wake word models that support it) by setting Snsr.LOW_FR_OPERATING_POINT. Adjust the duration of the low false-reject listening window with Snsr.DURATION_MS.

    • ‡ No Snsr.PARTIAL_RESULT_EVENT for empty hypotheses.
    • ‡ STT models start fewer threads.

        • Contact your Sensory Sales representative for additional languages.

  • Changed
    • The deprecated Snsr.SPOT_DELAY setting is read-only.
    • DSP library (for creating embedded spotter models) version 7.4.0.
    • ‡ The effects of : terminal rewrites are now visible in Snsr.RESULT_EVENT Snsr.RES_TEXT values.
    • ‡ The lightweight NLU processor scoring algorithm no longer considers :inserted and remove: words when ranking result parses. This potentially changes the primary NLU result, or the order in which NLU results are presented when Snsr.NLU_RES_MAX > 1, for grammars that use the : rewrite feature to better match expectations.
    • ‡ If the time required to compute a partial LVCSR result exceeds the Snsr.PUSH_DURATION_LIMIT, the Snsr.PARTIAL_RESULT_EVENT callback does not happen.
    • ‡ The Custom Recognition grammars support the literal use of special tokens by escaping them with a backslash.
  • Deprecated
    • Lightweight VAD. Use a Voice Activity Detector instead.
  • Removed
    • ‡ References to the <- slot assignment operator in Custom Recognition. This had been a symbol reserved for internal use only.
    • ‡ lvcsr-broad-enUS-3.1.0-t5.snsr
  • Fixed
    • Machine-learned Voice Activity Detector models could unexpectedly report an error that
      from-index (...) must be >= first-index (...)
    • A subset of spotter models showed increased false accept rates when used with Task Templates
    • ‡ tpl-opt-spot-vad-lvcsr-model mistakenly invoked Snsr.SILENCE_EVENT and Snsr.LISTEN_END_EVENT multiple times at end-of-stream if Snsr.SLOT_0 Snsr.INCLUDE_LEADING_SILENCE was set to @ 1.
    • ‡ tpl-spot-concurrent with a spotter model in Snsr.SLOT_0 and an LVCSR model (gated by a VAD, or spotter and VAD) in Snsr.SLOT_1 would mistakenly return Snsr.RES_NLU_SLOT_NAME and Snsr.RES_NLU_SLOT_VALUE values as NULL.
    • ‡ Snsr.NLU_SLOT_EVENT could still occur even if Snsr.COMPLETE_ONLY suppressed the recognition result.
    • ‡ Class-based recognition could fail with a segmentation fault when attempting to build a recognizer with too many class references.
  • Security
    • This sequence of operations could lead to SnsrSession.getString() returning an unterminated string:
      • Use SnsrSession.setStream() to set a configuration value to a sequence of bytes that does not include \0.
      • Save the model to persistent storage.
      • Load the modified model from persistent storage.
      • Access the configuration value with SnsrSession.getString().
    • SnsrStream.fromMemory() and SnsrStream.fromString() could potentially access the underlying byte[] memory after it was garbage-collected.
